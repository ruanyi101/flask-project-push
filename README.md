# 🔐 Flask Security Lab — OWASP Top 10 安全加固实战

> 基于 Python Flask 的用户信息管理平台，完整展示 **"漏洞发现 → Burp 验证 → 代码修复 → 复测确认"** 的安全开发闭环流程。

---

## 📋 项目简介

本项目围绕 **OWASP Top 10 (2021)** 安全框架，对一个 Flask 用户管理系统进行全流程安全加固。从源码白盒审计开始，逐项发现并修复了 15 个安全漏洞，覆盖密码存储、Session 管理、CSRF 防护、输入验证、访问控制、安全配置等关键领域。

## 🔧 修复措施

| 修复项 | 修复前 | 修复后 |
|-------|--------|--------|
| 密码存储 | 明文 `admin123` | PBKDF2-SHA256 加盐哈希 |
| Secret Key | 硬编码 `"dev-key-2025"` | 环境变量 / 随机 64 字符 |
| Session Cookie | 默认配置 | HttpOnly + SameSite=Strict + 30分钟超时 |
| CSRF | 无防护 | Token 校验 + SameSite 双重防护 |
| 暴力破解 | 无限制 | 同一 IP 60 秒限 5 次 |
| 用户枚举 | 不同错误提示 | 统一"用户名或密码错误" |
| Session Fixation | 登录后 ID 不变 | session.clear() 强制重置 |
| 密码回显 | `{{ user['password'] }}` | safe_user_info() 剥离 |
| 安全响应头 | 0 个 | 11 项（CSP/HSTS/XFO 等）|
| 输入验证 | 无校验 | 用户名 3~20 位，密码 6~128 位 |

## 🧪 Burp Suite 测试方法

| 测试项 | 方法 |
|-------|------|
| CSRF | Repeater 删 csrf_token → 403 |
| 暴力破解 | Intruder 发6次 → 第6次 429 |
| 用户枚举 | 不存在用户与密码错误提示一致 |
| Session 安全 | 查看 Set-Cookie 属性 |
| 密码泄露 | 搜索 password 不应出现在 HTML |
| 输入验证 | 超长/特殊字符 → 返回错误 |
